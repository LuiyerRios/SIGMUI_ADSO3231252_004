# Contributing to SIGMUI

Thank you for your interest in contributing to SIGMUI.

## Branching Strategy

Only the Scrum Master is authorized to create branches.

Main branches:

- main → Stable production version
- develop → Integration branch
- feature/* → New features

## How to Contribute

1. Pull the latest changes from main.
2. Request a branch from the Scrum Master.
3. Develop the assigned feature.
4. Commit changes with clear messages.
5. Submit a Pull Request for review.

## Commit Guidelines

- Use clear and descriptive commit messages.
- Follow separation of concerns.
- Do not upload sensitive files (.env).

## Code Standards

- Maintain clean architecture principles.
- Respect the three-layer structure.
- Ensure the application runs before pushing changes.

## Reporting Issues

If you find a bug or want to suggest an improvement, open an issue with a clear description.
